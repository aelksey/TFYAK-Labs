# Лабораторная работа №4: Построение LL(1) парсера

## Краткая теория

### 1. Что такое LL(1) парсер?

**LL(1)** — это нисходящий парсер, который:
- **L** — читает входную строку слева направо (Left-to-right)
- **L** — строит левый вывод (Leftmost derivation)
- **1** — использует один символ предпросмотра (lookahead)

### 2. Требования к LL(1) грамматике

Грамматика является LL(1), если выполняются следующие условия:

#### Условие 1: Отсутствие левой рекурсии
```
❌ A → Aα | β    (прямая левая рекурсия)
❌ A → Bα, B → Aβ (косвенная левая рекурсия)

✅ A → βA'
✅ A' → αA' | ε
```

#### Условие 2: Левая факторизация
```
❌ A → αβ₁ | αβ₂

✅ A → αA'
✅ A' → β₁ | β₂
```

#### Условие 3: Отсутствие конфликтов FIRST/FOLLOW для ε-правил
Для каждого правила `A → ε`:
```
FIRST(A) ∩ FOLLOW(A) = ∅
```

#### Условие 4: Отсутствие конфликтов FIRST/FIRST
Для каждого нетерминала `A → β₁ | β₂ | ... | βₙ`:
```
FIRST(βᵢ) ∩ FIRST(βⱼ) = ∅ для всех i ≠ j
```

### 3. FIRST и FOLLOW множества

#### FIRST(α) — множество терминалов, с которых может начинаться α

**Правила вычисления:**
1. Если α = ε → FIRST(α) = {ε}
2. Если α = a (терминал) → FIRST(α) = {a}
3. Если α = A (нетерминал) → FIRST(α) = FIRST(A)
4. Если α = A₁A₂...Aₙ:
   - Добавить FIRST(A₁)
   - Если ε ∈ FIRST(A₁), добавить FIRST(A₂), и т.д.

#### FOLLOW(A) — множество терминалов, которые могут следовать за A

**Правила вычисления:**
1. Если A — стартовый символ → добавить `$` (конец строки)
2. Если есть правило `B → αAβ`:
   - Добавить FIRST(β) \ {ε} в FOLLOW(A)
   - Если ε ∈ FIRST(β), добавить FOLLOW(B) в FOLLOW(A)
3. Если есть правило `B → αA` → добавить FOLLOW(B) в FOLLOW(A)

### 4. Алгоритм построения таблицы LL(1)

Для каждого правила `A → α`:

1. Для каждого терминала `a ∈ FIRST(α)`:
   - `TABLE[A, a] = A → α`

2. Если `ε ∈ FIRST(α)`:
   - Для каждого `b ∈ FOLLOW(A)`:
     - `TABLE[A, b] = A → α`

### 5. Типичные ошибки и их исправление

| Проблема | Пример | Исправление |
|----------|--------|-------------|
| Левая рекурсия | `E → E+T | T` | `E → TE'`, `E' → +TE' | ε` |
| Общий префикс | `S → if E S | if E S else S` | `S → if E S S'`, `S' → else S | ε` |
| Конфликт ε-правила | `A → ε`, `A → a` | Проверить FIRST(A) ∩ FOLLOW(A) |
| Двусмысленность | `S → if E S else S` | Переписать грамматику или добавить скобки |

### 6. Алгоритм устранения левой рекурсии

**Прямая левая рекурсия:**
```
A → Aα₁ | Aα₂ | ... | Aαₙ | β₁ | β₂ | ... | βₘ
```
где βᵢ не начинаются с A

**Преобразование:**
```
A  → β₁A' | β₂A' | ... | βₘA'
A' → α₁A' | α₂A' | ... | αₙA' | ε
```

### 7. Алгоритм левой факторизации

**Исходное:**
```
A → αβ₁ | αβ₂
```

**Преобразование:**
```
A  → αA'
A' → β₁ | β₂
```

### 8. Проверка LL(1) грамматики

**Алгоритм проверки:**
1. Устранить левую рекурсию
2. Провести левую факторизацию
3. Вычислить FIRST и FOLLOW для всех нетерминалов
4. Проверить для каждого правила `A → α₁ | α₂`:
   ```
   FIRST(α₁) ∩ FIRST(α₂) = ∅
   ```
5. Для каждого правила `A → ε`:
   ```
   FIRST(A) ∩ FOLLOW(A) = ∅
   ```

### 9. Пример построения LL(1) таблицы

**Грамматика:**
```
E  → TE'
E' → +TE' | ε
T  → FT'
T' → *FT' | ε
F  → (E) | id
```

**Таблица LL(1):**

| Нетерминал | id | + | * | ( | ) | $ |
|------------|----|----|----|----|----|----|
| E | E→TE' | | | E→TE' | | |
| E' | | E'→+TE' | | | E'→ε | E'→ε |
| T | T→FT' | | | T→FT' | | |
| T' | | T'→ε | T'→*FT' | | T'→ε | T'→ε |
| F | F→id | | | F→(E) | | |

### 11. Часто задаваемые вопросы

**Вопрос:** Всегда ли можно преобразовать грамматику к LL(1)?
**Ответ:** Нет. Некоторые языки не являются LL(1) (например, язык с неоднозначными конструкциями).

**Вопрос:** Как обрабатывать ε-правила?
**Ответ:** При построении таблицы для ε-правила заполняются все ячейки по FOLLOW множеству.

**Вопрос:** Что делать, если после преобразований грамматика остаётся не LL(1)?
**Ответ:** Возможно, потребуется изменить грамматику или использовать более мощный парсер (LL(k), LR(1)).

### 12. Памятка: признаки НЕ LL(1) грамматики

- ❌ Есть левая рекурсия (прямая или косвенная)
- ❌ Есть правила с общим префиксом
- ❌ Есть неоднозначность (dangling else)
- ❌ FIRST(A) ∩ FOLLOW(A) ≠ ∅ для какого-либо A → ε
- ❌ В таблице парсинга есть ячейка с несколькими правилами
