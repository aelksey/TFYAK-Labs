# Теория для защиты лабораторной работы №6

## «Синтаксис языков программирования. Преобразование транслируемой программы в постфиксную форму записи»

Ниже представлен полный теоретический материал, необходимый для успешной защиты лабораторной работы №6.

---

## 1. Основные понятия

### 1.1 Постфиксная форма записи (ПФЗ)

**Определение:** Постфиксная форма записи (обратная польская запись, RPN) — это способ записи выражений, при котором знак операции располагается **после** своих операндов.

| Форма записи | Пример | Описание |
|--------------|--------|----------|
| **Инфиксная** | `a + b` | Оператор между операндами |
| **Префиксная** (польская) | `+ a b` | Оператор перед операндами |
| **Постфиксная** (обратная польская) | `a b +` | Оператор после операндов |

### 1.2 История создания

- **1920-е годы** — Ян Лукасевич (польский логик) разработал бесскобочную запись
- **1950-е годы** — Чарльз Хэмблинг применил для компьютерных вычислений
- **1960-е годы** — Фридрих Бауэр и Клаус Самельсон использовали в алгоритмическом языке ALGOL
- **1970-е годы** — Hewlett-Packard внедрила в калькуляторы (HP-35)

### 1.3 Почему "постфиксная" и почему "обратная польская"

- **Постфиксная** — от лат. "post" (после) + "fixus" (прикреплённый) — оператор прикрепляется после операндов
- **Обратная польская** — в отличие от "прямой" польской записи (префиксной), где оператор стоит перед операндами

## 2. Свойства постфиксной записи

### 2.1 Основные свойства

| № | Свойство | Описание |
|---|----------|----------|
| 1 | **Бесскобочность** | Скобки не требуются — порядок операций определяется позицией операторов |
| 2 | **Однозначность** | Каждое выражение имеет единственную ПФЗ |
| 3 | **Обратимость** | По ПФЗ можно восстановить исходное выражение (со скобками) |
| 4 | **Компактность** | Обычно короче инфиксной записи |
| 5 | **Стековость** | Легко вычисляется с использованием одного стека |
| 6 | **Порядок операндов** | Совпадает с порядком в инфиксной записи |
| 7 | **Порядок операций** | Оператор появляется после обработки всех своих операндов |

### 2.2 Доказательство бесскобочности

В ПФЗ порядок выполнения операций полностью определяется взаимным расположением операторов и операндов, что можно доказать индукцией по глубине выражения:

- **База:** атомарное выражение (константа или идентификатор) не требует скобок
- **Шаг:** для выражения `A ○ B` в ПФЗ записывается `ПФЗ(A) ПФЗ(B) ○`. Поскольку `A` и `B` уже не требуют скобок, и оператор `○` применяется к результатам их вычисления, скобки не нужны

## 3. Способы преобразования в ПФЗ

### 3.1 Метод сортировочной станции (Дейкстра, 1961)

**Алгоритм:**

```
Инициализация: стек операторов S = ∅, выходная строка Q = ∅

Для каждого токена t в выражении слева направо:
    1. Если t — ОПЕРАНД:
        Добавить t в Q
    
    2. Если t — ОТКРЫВАЮЩАЯ СКОБКА '(':
        Поместить t в S
    
    3. Если t — ЗАКРЫВАЮЩАЯ СКОБКА ')':
        Пока вершина S != '(':
            Переместить оператор из S в Q
        Удалить '(' из S
    
    4. Если t — БИНАРНЫЙ ОПЕРАТОР o1:
        Пока S не пуст и на вершине S оператор o2 
        и (приоритет(o2) > приоритет(o1)) или 
           (приоритет(o2) == приоритет(o1) и o1 — левоассоциативный):
            Переместить o2 из S в Q
        Поместить o1 в S

Конец цикла

Пока S не пуст:
    Переместить оператор из S в Q
```

### 3.2 Стек операторов (упрощённый вариант для LL-грамматик)

Для LL(1)-грамматик преобразование можно выполнять непосредственно в процессе разбора:

```javascript
// Правила грамматики с действиями
expr : term
     | expr "+" term { emit("+"); }
     | expr "-" term { emit("-"); }
     ;

term : factor
     | term "*" factor { emit("*"); }
     | term "/" factor { emit("/"); }
     ;

factor : id { emit($id); }
       | const { emit($const); }
       | "(" expr ")"
       ;
```

### 3.3 Алгоритм дерева разбора (семантический)

1. Построить дерево разбора для выражения
2. Выполнить постфиксный (левый-правый-корень) обход дерева
3. При посещении узла-операнда — выдать его значение
4. При посещении узла-операции — сначала обойти поддеревья, затем выдать оператор

**Пример дерева для `a + b * c`:**
```
      (+)
     /   \
   (a)   (*)
         / \
       (b) (c)

Постфиксный обход: a, b, c, *, +
```

## 4. Преобразование управляющих конструкций

### 4.1 Общие принципы

Управляющие конструкции (условные операторы, циклы, switch) в отличие от выражений:
- Не могут быть представлены линейной последовательностью
- Требуют генерации **меток** и **операций перехода**
- Используют **стек меток** для обработки вложенности

### 4.2 Условный оператор (if-then-else)

**Схема ПФЗ:**

```
Условие метка_else JMPF
Блок_then
метка_выхода JMP
метка_else:
Блок_else
метка_выхода:
```

**Пример:** `if x > 0 then y = 1 else y = -1`

```
x 0 > L1 JMPF
y 1 =
L2 JMP
L1:
y -1 =
L2:
```

### 4.3 Цикл while

**Схема ПФЗ:**

```
метка_начало:
Условие метка_выход JMPF
Тело_цикла
метка_начало JMP
метка_выход:
```

**Пример:** `while i < 10 do i = i + 1`

```
L1:
i 10 < L2 JMPF
i i 1 + =
L1 JMP
L2:
```

### 4.4 Цикл for (с параметром)

**Схема ПФЗ:**

```
Инициализация_счётчика
метка_начало:
Проверка_условия метка_выход JMPF
Тело_цикла
Инкремент_счётчика
метка_начало JMP
метка_выход:
```

### 4.5 Оператор выбора (switch-case)

**Схема ПФЗ (каскад сравнений):**

```
Вычисление_выражения
dup (дублирование значения)
Сравнение_с_константой1
метка_case1 JMPT
Сравнение_с_константой2
метка_case2 JMPT
...
метка_default JMP

метка_case1:
Блок_case1
метка_выход JMP

метка_case2:
Блок_case2
метка_выход JMP

метка_default:
Блок_default
метка_выход:
```

## 5. Типы операций в ПФЗ

### 5.1 Вычислительные операции

| Тип | Примеры | Арность |
|-----|---------|---------|
| Арифметические | `+`, `-`, `*`, `/`, `%` | 2 |
| Унарные | `-`, `!`, `~` | 1 |
| Сравнения | `==`, `!=`, `<`, `>`, `<=`, `>=` | 2 |
| Логические | `&&`, `||` | 2 |

### 5.2 Управляющие операции

| Операция | Назначение | Арность |
|----------|------------|---------|
| `JMP` | Безусловный переход | 1 (метка) |
| `JMPT` | Переход, если вершина стека = true | 1 (метка) |
| `JMPF` | Переход, если вершина стека = false | 1 (метка) |
| `CALL` | Вызов функции | 1 (имя функции) |
| `RET` | Возврат из функции | 0 |
| `PUSH` | Поместить значение в стек | 1 (значение) |
| `POP` | Извлечь значение из стека | 0 |

### 5.3 Операции работы с памятью

| Операция | Назначение |
|----------|------------|
| `=` | Присваивание |
| `:=` | Присваивание с загрузкой адреса |
| `&` | Взятие адреса |
| `*` (в указателях) | Разыменование |

## 6. Алгоритм вычисления ПФЗ

### 6.1 Стековый алгоритм

```
Инициализация: стек S = ∅

Для каждого токена t в ПФЗ слева направо:
    Если t — ОПЕРАНД (число, переменная, константа):
        Поместить t в S
    
    Если t — БИНАРНЫЙ ОПЕРАТОР:
        b = извлечь верхний элемент из S  // правый операнд
        a = извлечь верхний элемент из S  // левый операнд
        результат = применить t к a и b
        поместить результат в S
    
    Если t — УНАРНЫЙ ОПЕРАТОР:
        a = извлечь верхний элемент из S
        результат = применить t к a
        поместить результат в S
    
    Если t — JMP, JMPT, JMPF, CALL, RET:
        // Специальная обработка для управления
        выполнить операцию перехода

Конец цикла

Результат = извлечь из S
```

### 6.2 Пример вычисления

**ПФЗ:** `3 4 5 * +`

| Шаг | Токен | Стек | Действие |
|-----|-------|------|----------|
| 1 | 3 | [3] | push(3) |
| 2 | 4 | [3,4] | push(4) |
| 3 | 5 | [3,4,5] | push(5) |
| 4 | * | [3,20] | pop(b=5), pop(a=4), push(4*5=20) |
| 5 | + | [23] | pop(b=20), pop(a=3), push(3+20=23) |

**Результат:** 23

## 7. Включение действий в грамматику

### 7.1 Принципы размещения действий

| Тип правила | Позиция действия | Пример |
|-------------|------------------|--------|
| Операнды (id, const) | Сразу после распознавания | `id { emit($id); }` |
| Бинарные операции | После обработки обоих операндов | `expr "+" term { emit("+"); }` |
| Унарные операции | После обработки операнда | `"-" factor { emit("~"); }` |
| Скобки | Не добавлять действий | `"(" expr ")"` |
| Присваивание | После левой и правой частей | `id "=" expr { emit("="); }` |

### 7.2 Действия для LL(1)-грамматик

```javascript
// Простые действия — только emit
id : [a-z]+ { emit($id); }
const : [0-9]+ { emit($const); }
plus : "+" { emit("+"); }
minus : "-" { emit("-"); }
mul : "*" { emit("*"); }
div : "/" { emit("/"); }

// Правила с действиями
expr : term
     | expr "+" term { emit("+"); }
     | expr "-" term { emit("-"); }
     ;

term : factor
     | term "*" factor { emit("*"); }
     | term "/" factor { emit("/"); }
     ;
```

### 7.3 Действия для LR(1)-грамматик

В LR-анализаторах действия выполняются при свёртке:

```javascript
expr : expr "+" term
{
    // При свёртке правила x + y -> expr
    emit("+");
}
```

### 7.4 Действия с метками

```javascript
// Для условного оператора
condition : "if" "(" expr ")" block elsePart
{
    var elseLabel = topLabel();
    var endLabel = newLabel();
    
    emit(elseLabel);
    emit("JMPF");
    
    emit(endLabel);
    emit("JMP");
    
    emit(elseLabel + ":");
    emit(endLabel + ":");
    
    popLabel();
}
```

## 8. Приоритеты и ассоциативность операций

### 8.1 Таблица приоритетов (от высшего к низшему)

| Приоритет | Операторы | Ассоциативность |
|-----------|-----------|-----------------|
| 1 (высший) | `()` (вызов функции) | Левая |
| 2 | `-` (унарный), `!`, `~` | Правая |
| 3 | `*`, `/`, `%` | Левая |
| 4 | `+`, `-` (бинарные) | Левая |
| 5 | `<`, `>`, `<=`, `>=` | Левая |
| 6 | `==`, `!=` | Левая |
| 7 | `&&` | Левая |
| 8 | `||` | Левая |
| 9 (низший) | `=` (присваивание) | Правая |

### 8.2 Как приоритеты влияют на ПФЗ

При преобразовании в ПФЗ операторы с более высоким приоритетом попадают в выходную строку **раньше** операторов с низким приоритетом.

**Пример:** `a + b * c`
- `*` имеет приоритет 3, `+` — приоритет 4
- `*` выполняется раньше → в ПФЗ `*` появляется раньше `+`
- Результат: `a b c * +`

### 8.3 Роль ассоциативности

**Левая ассоциативность** (`+`, `-`, `*`, `/`):

`a - b - c` → `a b - c -` (левоассоциативная обработка)

**Правая ассоциативность** (`=`):

`a = b = c` → `a b c = =` (правоассоциативная обработка)

## 9. Стек меток для вложенных конструкций

### 9.1 Зачем нужен стек меток

При обработке вложенных конструкций (например, условных операторов внутри других условных операторов) необходимо сохранять метки для корректной генерации переходов.

### 9.2 Реализация стека меток

```javascript
var labelStack = [];

function pushLabel(label) {
    labelStack.push(label);
}

function popLabel() {
    return labelStack.pop();
}

function topLabel() {
    return labelStack[labelStack.length - 1];
}
```

### 9.3 Пример использования для вложенных условий

```javascript
// Псевдокод для if (a) { if (b) { ... } else { ... } }
// Стек меток:
// - Для внешнего if: push(else1), push(end1)
// - Для внутреннего if: push(else2), push(end2)
// При закрытии внутреннего: pop() дважды
// При закрытии внешнего: pop() дважды
```

## 10. Типичные ошибки при реализации ПФЗ

| № | Ошибка | Последствие | Исправление |
|---|--------|-------------|-------------|
| 1 | Неправильный порядок операндов | `a b -` вместо `b a -` | Проверить порядок извлечения из стека |
| 2 | Двойная генерация операндов | Дублирование в ПФЗ | Добавлять операнды только в терминалах |
| 3 | Потеря операторов | ПФЗ не содержит операций | Проверить, что действия вызываются |
| 4 | Неправильные метки | Переходы на неверные адреса | Использовать стек меток |
| 5 | Не обработаны скобки | Нарушение приоритетов | Убедиться, что скобки не попадают в ПФЗ |
| 6 | Конфликт имён меток | Неоднозначность переходов | Использовать глобальный счётчик |
| 7 | Не учтена ассоциативность | Неправильный порядок при нескольких операциях | Правильно настроить условие в стеке |

## 11. Сравнение методов преобразования

### 11.1 LL(1) против LR(1) для ПФЗ

| Характеристика | LL(1) | LR(1) |
|----------------|-------|-------|
| **Направление обхода** | Сверху вниз | Снизу вверх |
| **Действия** | При разворачивании правил | При свёртке правил |
| **Левая рекурсия** | Недопустима | Допустима |
| **Простота действий** | Требуется аккуратность | Естественна для ПФЗ |
| **Позиция emit оператора** | После операндов (нужно ждать) | При свёртке (операнды уже обработаны) |

### 11.2 Рекомендации

- Для **арифметических выражений** удобнее **LL(1)** с методом сортировочной станции
- Для **полного языка** с управляющими конструкциями удобнее **LR(1)** из-за естественной обработки левой рекурсии

## 13. Формулы и определения (шпаргалка)

### Определения

```
ПФЗ(операнд) = операнд
ПФЗ(A ○ B) = ПФЗ(A) ПФЗ(B) ○
ПФЗ(○ A) = ПФЗ(A) ○ (для унарного оператора)
```

### Преобразование выражений

```
a + b        → a b +
a - b        → a b -
a * b        → a b *
a / b        → a b /
-a           → a ~ или a -
a && b       → a b &&
a || b       → a b ||
a == b       → a b ==
a != b       → a b !=
a < b        → a b <
a > b        → a b >
a <= b       → a b <=
a >= b       → a b >=
```

### Метки и переходы

```
метка:               → LABEL:
переход на метку     → LABEL JMP
условный переход     → LABEL JMPF (если условие ложно)
                     → LABEL JMPT (если условие истинно)
```
